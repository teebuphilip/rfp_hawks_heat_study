Methodology

This page spells out the full FMV to wins chain, the totals columns, and the proxy diagnostics shown on the live pages.

FMV FMVW HBB Proxy totals

Stage Order

1. FMV Cap-constrained market value estimated from the simulation layer.
2. Wins Eq Convert salary-equivalent FMV into a wins-equivalent estimate.
3. HBB Factor Apply the team-context multiplier to the wins bridge.
4. HBB Wins HBB-adjusted wins before any proxy correction.
5. Proxy Delta Apply the offense/defense proxy correction as a separate total.
6. Proxy Wins Final proxy-adjusted wins after the correction is added to HBB wins.
7. Proxy Read Show offense proxy, defense proxy, turnovers, and FTA as the diagnostic readout.

Totals Columns

Column Meaning
FMV Salary-equivalent estimate from the simulation output.
Wins Eq FMV translated into a wins-equivalent number using the bridge slope.
HBB Team-context factor applied to the wins bridge.
HBB Wins Wins Eq after HBB is applied.
Proxy Δ Additional correction from the offense / defense proxy layer.
Proxy Wins HBB Wins plus Proxy Δ.
Proxy Read Offense proxy, defense proxy, turnovers, and FTA as the explanation layer.

Plain-English Flow

A Start with the teammate-neutral FMV salary estimate.
B Convert that salary into wins-equivalent output.
C Adjust the wins output for team context using HBB.
D Apply the proxy delta as a visible correction term.
E Show the proxy read separately so the arithmetic is auditable.
