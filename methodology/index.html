End-to-End Flow

1. Start with the projection engine

The engine produces a season projection for each player. Core fields include minutes, points, rebounds, assists, steals, blocks, turnovers, FG%, FT%, three-point percentage, fantasy points, ceiling, floor, and consistency. In the engine output, FTA is also available and can be used as a pressure signal.

2. Load the FMV comparison pool

The FMVW page uses a fixed top-200 pool for the 25-26 comparison surface. That pool provides the market-value side of the comparison: player name, team, position, salary-equivalent FMV, and band label. The live page also loads the 26-27 projection bundle and the league table so the comparison can read proxy and HBB values at the same time.

3. Normalize the player signal

Player output is adjusted toward a common roster context so surrounding teammates do not dominate the evaluation. This is the teammate-neutral layer. The idea is to isolate player value before market pricing happens.

4. Price FMV

The FMV layer turns the teammate-neutral player signal into a salary-equivalent estimate using a small cap-constrained Monte Carlo simulation. It is a model-implied market value, not an observed contract price.

5. Bridge FMV to wins

The FMVW bridge converts salary-equivalent value into wins-equivalent output using the current replacement-level and salary-per-win constants. This gives a roster-level way to compare player value in season terms rather than only dollars.

6. Apply HBB

HBB is a team-context multiplier read from the league table. It shifts the wins-equivalent value up or down depending on the surrounding roster, balance, support, and pressure environment.

7. Apply the proxy layer

The current proxy layer uses box-score approximations to add an offense-first correction and a separate defense correction. The proxy layer is not the same thing as the FMVW bridge. It sits on top of it.

8. Produce proxy-adjusted wins

The current headline number is the FMVW win total after the proxy correction is applied. That is the number shown on the FMVW page when you compare two players.

Current Configuration

Projection source

Loaded from the live projection bundle, which carries minutes, scoring, rebounding, assist, turnover, shooting, and FTA fields.

FMV pool

Fixed 25-26 top-200 player pool used for the comparison surface.

FMVW bridge constants

Replacement salary is $2,000,000. Salary per win is $4,000,000. Those constants convert FMV dollars into the wins-equivalent bridge before HBB is applied.

HBB

Applied as a team-level multiplier after FMVW is computed. The current league bundle carries a league HBB baseline and a team HBB factor for each team. The current league HBB baseline in the live bundle is 0.679.

Position normalization

Proxy z-scores are standardized within position groups so a guard is compared to guards, a wing to wings, and a big to bigs before the headline gap is shown.

Proxy wins weights

Offense weight is compressed with a soft cap, while weak defense is penalized more than strong defense is rewarded. The exact constants are configurable in code.

Current proxy weights

Offense proxy uses points, assists, threes, FTA, FG%, FT%, and turnovers. Defense proxy uses rebounds, steals, blocks, and turnovers. The proxy layer is intentionally box-score based for now.

Three-point input

The current proxy uses the live feed's three-point field as a shooting signal, not as a shot-chart volume model. It is a proxy input, not a full tracking metric.

Current Formula

Offense proxy

offense_proxy_raw = 2.0*points + 1.5*assists + 1.0*three_pt_pct + 1.5*fta + 0.75*fg_pct + 0.5*ft_pct - 1.0*turnovers

Defense proxy

defense_proxy_raw = 0.1*rebounds + 3.0*steals + 5.5*blocks - 2.0*turnovers

Bridge formula

wins_equivalent = (sim_mean_salary - replacement_salary) / salary_per_win

hbb_wins = wins_equivalent * hbb_factor

proxy_adjusted_wins = hbb_wins + proxyWinsAdjustment(offense_proxy_z, defense_proxy_z)

Proxy adjustment to wins

The offense proxy is compressed with a soft cap. The defense proxy is asymmetric: positive defense helps, but weak defense hurts more than strong defense helps. That adjustment is added to the HBB-adjusted FMVW result to produce proxy-adjusted wins.

Current code defaults: offense weight 0.18, offense cap 1.35, defense positive weight 0.34, defense negative weight 0.69.

Configuration Items

Offense proxy weights Points 2.0, assists 1.5, threes 1.0, FTA 1.5, FG% 0.75, FT% 0.5, turnovers -1.0.
Defense proxy weights Rebounds 0.1, steals 3.0, blocks 5.5, turnovers -2.0.
Proxy wins config Offense weight 0.18, offense cap 1.35, defense positive weight 0.34, defense negative weight 0.69.
HBB Team-context multiplier applied after the FMVW bridge and before the proxy adjustment. Current live values come from the league table file and the team HBB factor in that file is the one applied to each player.
League HBB baseline The live league bundle currently reports 0.679 as the league HBB baseline.
FTA handling FTA is treated as a pressure signal on offense. It increases the offensive proxy because it forces the defense into contact, fouls, and rotation stress.
Turnovers Turnovers penalize the offensive proxy and also reduce the defensive proxy slightly because they are possession losses, not impact events.
Three-point field The current live proxy uses the feed's three-point field as an input signal. It is configurable, but the present implementation treats it as part of the offense proxy rather than a separate shot-chart model.

How To Read The Outputs

FMV

Salary-equivalent market price.

Wins Eq

FMV translated to wins using the current salary-per-win bridge.

HBB Wins

Wins-equivalent value after the team-context multiplier is applied.

Proxy Wins

Wins after the offense-first proxy correction is layered on top of HBB Wins.

Off Proxy / Def Proxy

Diagnostic z-scores that show how the player compares to the positional peer group on the current proxy surface.

Known Limits

Not a final defensive model

The current live page still uses box-score defense proxies. It does not yet have full shot-defense / rim-protection tracking in the browser layer.

FTA is now present, but still a proxy input

FTA is available in the engine projection layer and now in the live FMVW bundle, but it still represents a pressure proxy rather than a full foul-drawing model.

Future layer

The next advanced step is a richer defensive history layer with opponent defense data, archetype clustering, and championship DNA analysis.
